# 질문 유형

Skillnote AI는 TIL 포스트의 내용을 분석하여 5가지 유형의 문제를 자동으로 생성합니다. 각 유형은 다양한 방식으로 지식을 테스트하여 학습 효과를 극대화합니다.

---

## 문제 카드 공통 구성

모든 문제 카드에는 다음 정보가 포함됩니다.

- **문제 유형 배지**: 문제 유형을 한눈에 구분
- **난이도**: 1~5단계로 표시 (1: 쉬움, 5: 어려움)
- **출처 포스트 제목**: 문제가 생성된 TIL 포스트
- **세션 진행 상황**: 현재 몇 번째 문제인지 카드 상단에 표시

---

## 유형 1: O/X (참/거짓)

주어진 문장이 맞는지 틀리는지를 판단하는 문제입니다.

**답변 방법**: **O (참)** 또는 **X (거짓)** 버튼을 클릭합니다.

**채점 방식**: 자동 채점 (즉시 정답과 비교)

**예시**
```
JavaScript에서 let으로 선언한 변수는 재선언이 가능하다.
→ X (거짓)
```

---

## 유형 2: 객관식 (Multiple Choice)

보기 중에서 가장 알맞은 답을 하나 선택하는 문제입니다.

**답변 방법**: **A, B, C, D** 중 하나의 보기를 클릭합니다.

**채점 방식**: 자동 채점 (즉시 정답과 비교)

**예시**
```
다음 중 CSS Flexbox에서 주축(main axis)을 설정하는 속성은?

A. align-items
B. flex-direction   ← 정답
C. justify-self
D. flex-wrap
```

---

## 유형 3: 빈칸 채우기 (Fill in the Blank)

문장에서 빠진 단어나 구문을 직접 입력하는 문제입니다. 하나의 문제에 빈칸이 여러 개일 수 있습니다.

**답변 방법**: 각 빈칸에 알맞은 내용을 텍스트로 입력합니다.

**채점 방식**: 자동 채점 (대소문자 구분 없이 비교)

> **Tip:** 빈칸 채우기 문제는 대소문자를 구분하지 않습니다. `react`와 `React`는 동일하게 처리됩니다.

**예시**
```
HTTP 메서드 중 리소스를 생성할 때는 [___], 수정할 때는 [___]을 사용한다.
→ POST, PUT
```

## 유형 4: 단답형 (Short Answer)

핵심 개념을 짧은 문장으로 설명하는 문제입니다.

**답변 방법**: 텍스트 입력 창에 답변을 작성합니다. 최소/최대 글자 수 제한이 있습니다.

**채점 방식**: AI 채점 (Lambda 함수를 통해 의미적으로 평가)

> **Tip:** 단답형은 완벽한 문장이 아니어도 됩니다. 핵심 키워드와 개념을 포함하면 정답으로 인정될 수 있습니다.

**예시**
```
클로저(Closure)란 무엇인가요?
→ 함수가 선언될 때의 스코프를 기억하여 외부 함수가 종료된 후에도 외부 변수에 접근할 수 있는 함수
```

---

## 유형 5: 서술형 (Long Answer)

특정 개념이나 주제를 상세하게 서술하는 문제입니다.

**답변 방법**: 텍스트 입력 창에 에세이 형식으로 상세히 작성합니다. 최소/최대 글자 수 제한이 있습니다.

**채점 방식**: AI 채점 (Lambda 함수를 통해 평가, 0~100점 점수 부여)

> **Tip:** 서술형 문제는 점수(0~100)와 함께 AI의 상세한 피드백을 받을 수 있습니다. 틀렸더라도 피드백을 통해 정확한 개념을 확인하세요.

**예시**
```
RESTful API 설계 원칙에 대해 설명하고, 실제 적용 예시를 들어보세요.
→ (상세한 서술형 답변 작성)
```

## 유형별 비교 요약

| 유형 | 답변 방식 | 채점 방식 | 점수 제공 |
|------|-----------|-----------|-----------|
| O/X (참/거짓) | 버튼 클릭 | 자동 채점 | 정답/오답 |
| 객관식 | 보기 선택 | 자동 채점 | 정답/오답 |
| 빈칸 채우기 | 텍스트 입력 | 자동 채점 | 정답/오답 |
| 단답형 | 텍스트 입력 | AI 채점 | 정답/오답 |
| 서술형 | 텍스트 입력 | AI 채점 | 0~100점 |
